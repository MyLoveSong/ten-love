## Why

The repository currently contains a Vue/Vite frontend demo and documentation that frames the project as a full-stack health-monitoring application. The current research goal is to produce credible experiments, evidence, and manuscript-ready claim boundaries. The frontend does not support the core experimental claims and increases the active maintenance surface.

## What Changes

- Remove active frontend source from the GitHub repository.
- Remove active frontend source from the local `/home/data/xzy/system` source tree.
- Update README, audit, tracking, and manuscript draft language so active scope is research code, data governance, result evidence, and reproducibility.
- Keep provenance through Git history and this OpenSpec change instead of adding an archive directory.

## Capabilities

### New Capabilities
- `research-repo-governance`: Defines the active repository boundary for research assets, excluded frontend surfaces, and claim-safe documentation.

### Modified Capabilities

None. This project did not previously contain an OpenSpec capability baseline.

## Impact

Affected areas:
- `frontend/` removed from active GitHub scope.
- `/home/data/xzy/system/frontend/` and `/home/data/xzy/system/apps/frontend/` removed from local active source.
- README, audit, tracking, and manuscript draft wording updated.

No raw data, model checkpoints, generated experiment outputs, or large datasets are moved or deleted.
