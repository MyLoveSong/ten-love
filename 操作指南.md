## 🛠️ 环境准备

### 1.1 系统要求
- **操作系统**: Windows 10/11, macOS, Linux
- **Python版本**: 3.8 或更高版本
- **内存**: 建议 8GB 以上
- **存储空间**: 至少 2GB 可用空间

### 1.2 安装依赖
```bash
# 进入项目目录
cd "创新创业/血糖预测/x血糖"

# 安装所有依赖包
pip install -r requirements.txt

# 如果安装速度慢，可以使用国内镜像
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple/
```

### 1.3 验证安装
```bash
# 运行快速启动脚本验证环境
python quick_start.py
```

## 🔧 功能模块使用

### 3.1 基础模型 (blood2.py)
```bash
# 直接运行基础模型
python blood2.py
```

**功能**: 使用传统机器学习算法进行血糖预测
**适用场景**: 快速验证、小数据集测试
**输出**: 预测结果和模型性能指标

### 3.2 GluFormer模型 (gluformer_model.py)
```bash
# 运行深度学习模型
python gluformer_model.py
```

**功能**: 使用LSTM+GRU融合的深度学习模型
**适用场景**: 高精度预测、大数据集
**输出**: 训练过程、模型权重、性能评估

### 3.3 多模态特征提取 (multimodal_features.py)
```bash
# 运行多模态特征提取
python multimodal_features.py
```

**功能**: 处理图像、文本、数值多模态数据
**适用场景**: 复杂数据输入、图像识别
**输出**: 特征向量、融合结果

## 🌐 API服务使用

### 4.1 启动API服务
```bash
python api_service.py
```

**服务地址**: http://localhost:8000
**API文档**: http://localhost:8000/docs
**健康检查**: http://localhost:8000/health

### 4.2 主要API接口

#### 4.2.1 血糖预测
```bash
POST http://localhost:8000/predict/glucose
Content-Type: application/json

{
    "age": 45,
    "bmi": 24.5,
    "fasting_glucose": 5.6,
    "postprandial_glucose": 7.8,
    "hba1c": 6.2,
    "insulin": 15,
    "cholesterol": 180,
    "ldl": 110,
    "hdl": 50,
    "triglycerides": 150,
    "physical_activity": 2,
    "sleep_quality": 3,
    "stress_level": 2,
    "diabetes_type": 2,
    "pregnant": 0
}
```

#### 4.2.2 食物推荐
```bash
POST http://localhost:8000/recommend/food
Content-Type: application/x-www-form-urlencoded

glucose_level=8.5&cultural_preferences=清真&meal_type=lunch
```

#### 4.2.3 多模态预测
```bash
POST http://localhost:8000/predict/multimodal
Content-Type: multipart/form-data

numerical_data: {"age": 45, "bmi": 24.5, ...}
image: [图片文件]
food_description: "米饭和青菜"
```

### 4.3 测试API

## 🔍 故障排除

### 7.1 常见问题

#### 问题1: 依赖包安装失败
```bash
# 解决方案1: 使用国内镜像
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple/

# 解决方案2: 升级pip
python -m pip install --upgrade pip

# 解决方案3: 使用虚拟环境
python -m venv venv
venv\Scripts\activate  # Windows
pip install -r requirements.txt
```

#### 问题2: 模型训练失败
```bash
# 检查数据文件是否存在
ls *.csv

# 检查内存使用
# 如果内存不足，减少batch_size或使用更小的数据集
```

#### 问题3: API服务启动失败
```bash
# 检查端口是否被占用
netstat -ano | findstr :8000

# 使用不同端口
python api_service.py --port 8001
```

#### 问题4: 导入错误
```bash
# 运行修复脚本
python fix_imports.py

# 重新安装问题包
pip install opencv-python transformers
```

### 7.2 性能优化

#### 内存优化
- 使用较小的batch_size
- 减少模型层数
- 使用数据生成器

#### 速度优化
- 使用GPU加速（如果可用）
- 减少数据预处理步骤
- 使用更简单的模型
