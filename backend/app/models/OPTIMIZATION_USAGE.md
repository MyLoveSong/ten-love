# 推荐系统参数优化器使用说明

## 概述

推荐参数优化器应用了启发式算法思想（参考田忌赛马、螳螂虾等优化算法），自适应优化推荐系统的关键参数，提升推荐效果。

## 优化目标

### 1. 口味权重优化
- **目标**：优化6个口味因子的权重（辣、甜、酸、咸、鲜、苦）
- **方法**：基于用户偏好和食物样本，自适应调整权重
- **参考算法**：田忌赛马优化算法（在关键因子上投入更多资源）

### 2. 置信度权重优化
- **目标**：优化置信度计算的4个因子权重（文化适配、营养平衡、档案完整性、数据充足度）
- **方法**：基于用户档案样本，自适应调整权重
- **参考算法**：螳螂虾优化算法（快速定位重要因子）

### 3. 多目标优化
- **目标**：平衡多个目标（文化适配度、营养平衡、用户满意度）
- **方法**：使用加权求和法，参考田忌赛马策略比较不同参数组合
- **参考算法**：田忌赛马优化算法（比较策略思想）

## 使用方法

### 启用优化器

```python
from app.services.integrated_diabetes_service import IntegratedDiabetesService

# 创建服务时启用自适应优化
service = IntegratedDiabetesService(
    use_adaptive_optimization=True  # 启用自适应优化
)
```

### 自动优化

优化器会在以下场景自动优化参数：

1. **口味权重优化**：在计算文化适配度时，根据用户历史膳食记录自动优化口味权重
2. **置信度权重优化**：在计算推荐置信度时，根据用户档案自动优化置信度权重

### 手动优化

```python
# 获取优化后的参数
optimized_params = service.recommendation_optimizer.optimize_all_parameters(
    user_data={
        'cultural_preferences': {
            'spice_tolerance': 0.8,
            'sweet_preference': 0.3,
            # ...
        }
    },
    food_samples=[...],
    user_profile_samples=[...]
)

# 使用优化后的参数
flavor_weights = optimized_params['flavor_weights']
confidence_weights = optimized_params['confidence_weights']
```

## 算法思想

### 田忌赛马优化算法思想

核心策略：**在关键因子上投入更多资源**

- 高重要性因子（重要性>20%）：更激进地调整权重（80%重要性 + 20%初始权重）
- 低重要性因子（重要性≤20%）：保守地调整权重（60%重要性 + 40%初始权重）

### 螳螂虾优化算法思想

核心特性：**快速定位重要因子**

- 计算因子的区分度（方差）
- 计算因子的匹配度（与用户偏好的匹配）
- 计算因子的偏好强度（偏离中性值的程度）
- 综合三个维度快速识别关键因子

### 自适应加权优化

核心方法：**基于数据动态调整权重**

- 重要性得分 = 40%区分度 + 30%匹配度 + 30%偏好强度
- 权重调整 = 重要性得分 × 调整系数 + 初始权重 × (1 - 调整系数)

## 性能优化

1. **缓存机制**：优化后的参数会被缓存，避免重复计算
2. **增量更新**：仅在有足够数据时进行优化
3. **降级策略**：优化器不可用时自动使用固定权重

## 复杂度分析

- **时间复杂度**：O(n × m)，n为样本数，m为因子数
- **空间复杂度**：O(n + m)，缓存和样本数据
- **实际影响**：优化过程在后台异步执行，不影响推荐响应时间
